  简单了解一下数组的相关内容，在JAVA中，数组是集合的基础。但是在API中，并没有数组的相关文档。只有java.util.Arrays，但它明显不是数组的API文档。
使用如下测试代码去查询数组的类：

public class Main {   
  public static void main(String args[]){   
    int a[] = new int[10]; 
    Class clazz = a.getClass();   
    System.out.println(clazz.getName());   
  }   
}

输出结果为 [I
  说明数组类的类名为 [I这明显不符合JAVA类的命名规范，[ 也是java中的非法标识符。
通过如下代码测试一下数组类的成员变量、成员方法等：

public class Main {   
  public static void main(String[] args) {   
    int a[] = new int[10]; 
    Class clazz = a.getClass();   
    System.out.println(clazz.getDeclaredFields().length); //属性
    System.out.println(clazz.getDeclaredMethods().length); //方法   
     //构造方法  
    System.out.println(clazz.getDeclaredConstructors().length);
     //注释 
    System.out.println(clazz.getDeclaredAnnotations().length);
     //Class对象  
    System.out.println(clazz.getDeclaredClasses().length);
    System.out.println(clazz.getSuperclass()); //父类
  }   
} 

显示结果为：

0 
0

0

class java.lang.Object  

  从测试结果来看，数组类的父类是Object类，但是它没有成员变量和成员方法、构造方法等，连注释都没有。但是我们在使用数组的时候，却明确的可以使用.length属性来获取到数组的长度！
  这里并没有实力去看JVM的源代码，查阅网上相关资料内容如下：数组类应该是JVM虚拟机自己创建了[I类，在《JAVA虚拟机规范》(The JavaTM Virtual Machine Specification  Second Edition)中对其进行了严谨的规范。简单概括：
  类加载器先看看数组类是否已经被创建了。如果没有，那就说明需要创建数组类；如果有，那就无需创建了。
  如果数组元素是引用类型，那么类加载器首先去加载数组元素的类。
  JVM根据元素类型和维度，创建相应的数组类。

  通过字节码文件查看创建过程：

public class Main {   
public static void main(String[] args)   
{ int a[] = new int[2]; int i = a.length;   
}   
}

1 newarray 10 (int)    //将2弹出操作数栈，作为长度，创建一个元素类型为int, 维度为1的数组，并将数组的引用压入操作数栈

数组的定义:
元素类型[] 数组名 = new 元素类型[元素个数或数组长度];
例：int[] arr = new int[5]; 
使用[]来标识数组类(不晓得这里[]和数组类的类名[I是不是一个巧合)

 new 用来在内存中开辟空间。
  5 在这里用于指定在堆内存中开辟空间的大小。

二维数组：实质就是存储的是一维数组。
数组定义：
数组类型[][] 数组名 = new 数组类型[一维数组的个数][每一个一维数组中元素的个数];
例：int[][] arr = new int[5][6]; 
arr指向一个长度为5的一维数组，一维数组的每个元素保存一个新的长度为6的一维数组的内存地址。

数组的维度最大为255

JAVA在util中提供了java.util.Arrays类对数组进行操作。
Arrays类直接继承Object类，专门用于操作数组，主要提供排序，搜索方法。此类还包含一个允许将数组作为列表来查看的静态工厂。
主要关注一下指定比较器的sort方法即可，其余参考API。
